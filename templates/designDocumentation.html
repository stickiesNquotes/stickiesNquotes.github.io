Abstract

The purpose of this project was to demonstrate my knowledge of HTML, CSS, and JavaScript (including jQuery) through the development and design of an interface and several webpages tied together in one website. Throughout this project, I started to understand the development of webpages and how changing one element or forgetting to close a tag can change the whole feel and design of the page. In the development of cookies and my interfaces, I continued my learning of JavaScript through variables, functions, and cookies. Overall, I am really happy with how the website turned out and seeing the design concepts I had from the start of the project to the end of it because it showed me how much trial and error development goes into creating a functioning website with user interactivity.

Design Goals

  1. To build an interactive interface which can be used to create sticky note reminders and an online journal.
  2. To use LocalStorage and Cookies to retain data between pages and sessions.
  3. To use javascript as the basis for allowing the user to interact with the pages.

Design Concept

One of the primary design concepts was continuity across the main pages.

JavaScript

  1. The buttons in the upper right hand corner that let the user chose the background use some jQuery inorder to set the cookie from the buttons:
      $("#bkgrnd-bboard").click(function(){
         document.getElementById("content").style.backgroundImage = "url(images/chalkboard.png)";
         setCookie("bkgrnd", "url(images/chalkboard.png)");
      });
  2. The saveNotes() function is located in the myLocalStore.js. It was probably one of the harder functions to write out of the whole project. It utilizes local storage to save the notes for the location, color, and content of the note
      saveNotes = function(){
        var noteList = document.getElementsByClassName("jStickyNote");
        var noteNum;
        var noteobj;
        var noteArr = [];
        for (noteNum = 0; noteNum < noteList.length; noteNum++) {
          noteObj = new Object();
          noteObj["top"] = noteList[noteNum].getBoundingClientRect().top.toString() + "px";
          noteObj["left"] = noteList[noteNum].getBoundingClientRect().left.toString() + "px";
          noteObj["color"] = noteList[noteNum].parentNode.getAttribute('class').split(' ')[0];
          noteObj["content"] = noteList[noteNum].getElementsByTagName("TEXTAREA")[0].value;
          noteArr.push(noteObj);
        }
        localStorage.setItem('NoteList', JSON.stringify(noteArr));
      }
  3. On every page there is an internal script to run different functions from several .js files. The script checks the cookies and updates the clock information:
      window.onload=function(){

Testing

Testing was performed by my father throughout the creation of the website. He pointed out inconsistencies in funtion and improvements for user interaction. Testing led to several redesigns and changes in the approach from how I originally envisioned the pages.

Alpha

  1. Using github was challenging. It took some time to get used to the limitations of the interface.
  2. Researching examples of sticky notes led me down a few wrong paths. I could not find an example that did everything I wanted, so I ended up creating my own. This in turn led to learning simple jQuery.
  3. Mixing standard javascript and jQuery. I am familiar with Java. Javascript is a bit different for me, and jQuery can be downright confusing.

Beta

  1. Changes to background were not carrying over from page to page. This ended up being inconsistency between pages.
  2. Original sticky notes all stacked in same spot and were difficult to drag due to small side handles.
  3. Ghosts of notes and journal pages. Originally, each note and journal page was entered into LocalStorage as separate items. This made it difficult when saving information for page changes. After a bit of research, I found JSON Stringify and Parse solution which simplified the whole storage issue.

Maker's Mark Video

The transcript for the video:
When on the main index page, we can click on the Guest at the top of the page to change the name. When you click okay or hit enter after inputting a name, the username is stored into a cookie and this cookie is used across multiple pages on the website. You can only change the username on the index page.

Continuing with cookies, you can click on any of these background buttons to change the background of the page. The default is set to be a corkboard, but once you click on any button, it sets a cookie for the background and calls that cookie for the background throughout the website. For example, I can click on the black board button to set and save my background as a blackboard.

When the body first loads, it will check to see if there is a cookie saved for both the username and the background. If it returns true for the username, it will get the username and set it to be at the top of the page. If it returns false, the username will be set to "Guest" at the top of the page. If the cookie returns true for the background, it will get the background image and set it to be the correct background. If the background cookie returns false, it will set it to the default corkboard. Unlike the username, you can change the background image from any page on the site.

The cookies integrated on this website are in the form of a JavaScript .js file.

Underneath the background buttons, there is the date and clock function. This data is accessed from calling the Date() function that comes along with JavaScript.

From the main page, we can access the sticky note interface, journal interface, and the project documentation page.

If we go to the Sticky Note Board, we can see that the username and background have been checked and loaded into the correct position. On the sticky note dashboard, we can double click to get a blue sticky note, or click any four of these buttons to spawn in a new sticky note. I can move the sticky note to anywhere I would like and write content to the sticky note. Now, it would not be very helpful if the reminders diasppeared when we came back to the page at a later time. So, when we close out the page, the sticky notes are stored into local storage when the body is unloaded. When we revisit the page and load the body, the sticky notes are in the same position with the correct text as we left them. We can re-edit the sticky notes, and move them again as we please. If I want to get rid of a sticky note, all I have to do is click on this X button in the top left hand corner. Goodbye reminder.

Now, if we work our way back to the main page, and go to the journal entry page, we can store a local online journal. The style behind the journal is meant to look as though we are actually writing a journal entry. The background image is a barely crumpled piece of paper. If we click new entry, and we can start typing in the text area. Unlike the sticky notes, you cannot go back and re-edit the journal entry. In order to save the journal entry, we just need to click on new entry or refresh the page. There is currently no way to remove a single journal entry, so if you want to clear the journal, you have to go back to the main page, go into developer tools, find local storage, and delete the journalEntry storage item.

If we go look at the project information page, we can look at the design concept heirarchy tree and see what pages access different site pages, JavaScript and CSS pages. The header.css file is used throughout the site to create the teal color header with the username, background buttons, and time and clock. Each page on the site has it's own external CSS file for easy access to change any style. Every site page also calls the clock and cookie javascript files. And the interfaces call their respective javascript files.
